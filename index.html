Webpage Layout

A weboldal kinézete a projekt befejezéséig még változhat, ha ez megtörténne, a dokumentációs oldal frissülni fog.

Főoldal

A felhasználó a weboldalon először a főoldalra érkezik, itt egy összefoglalót kap arról, miről is szól az oldalunk. Emiatt itt különféle érdekességeket és akciókat találhat, aki meglátogatja az oldalt.

Webshop

A webshopban található meg a készleten lévő összes termék. Eleinte az összes megjelenik, viszont lehet pontosabb keresést is folytatni.

Kosár

Amikor a felhasználónak tetszik egy termék és meg akarja vásárolni, kijelölheti, hogy mennyit szeretne belőle venni, és beteheti a kosarába. Ezek a kosárban lévő termékek rendelésig vagy az oldalról való kilépésig benne maradnak. A kosárban a vásárló saját adatait beírva meg tudja rendelni a kosárba összeválogatott termékeket.

Termékkategóriák

Ruhák

Fiú és lány ruhákra bontva megtalálhatóak a webshopon elérhető baba öltözékek, méretekre bontva.

Játékok

A ruhákhoz hasonlóan a játékok is fiú és lány kategóriákra bontva találhatóak meg, viszont pár játék megtalálható mindkettő csoportban.

Ételek

Az ételek kategóriában külön csoportosítás nélkül lehet megtalálni minden ételt, ami a weboldalon található.

Alapból minden termék megjelenik a boltban, de különféle szűrőkkel le lehet szűkíteni a keresést. A szűrőkről lejjebb, [ITT] tudhat meg többet.
A termékeket egy adatbázisban tároljuk. Az adatbázis diagramja lejjebb, [ITT] tekinthető meg.
A felhasználó beteheti a termékeket egy kosárba, amiket megvásárolni készül. A vásárlásról [ITT] tájékozódhat.

Adatbázis

A kódon felül az adatbázis a teljes projekt gerince. Az adatok MySQL-ben vannak tárolva, és folyamatos kapcsolatban áll a weboldallal

DBDiagramban ábrázolt minta az adatbázisról, amit MySQL segítségével tárol az oldal

                                        CREATE TABLE `category` (
                                          `id` int PRIMARY KEY,
                                          `name` string
                                        );

                                        CREATE TABLE `product` (
                                          `id` int PRIMARY KEY,
                                          `categoryID` int,
                                          `name` string,
                                          `price` float,
                                          `orderAmount` int,
                                          `description` string,
                                          `weight` string,
                                          `brand` string,
                                          `color` string,
                                          `madeIn` string,
                                          `gender` char(1),
                                          `cSize` string,
                                          `stock` int,
                                          `image` string
                                        );

                                        CREATE TABLE `order` (
                                          `orderID` int PRIMARY KEY,
                                          `productID` int,
                                          `uFirstName` string,
                                          `uLastName` string,
                                          `uEmail` string,
                                          `u1AddressLine` string,
                                          `u2AddressLine` string,
                                          `uPhone` string
                                        );

                                        ALTER TABLE `product` ADD FOREIGN KEY (`categoryID`) REFERENCES `category` (`id`);

                                        ALTER TABLE `order` ADD FOREIGN KEY (`productID`) REFERENCES `product` (`id`);

Az előbb ábrázolt DBDiabram MySQL kód megfelelője, a rendes programban ez - vagy egy ehhez nagyon hasonló - kód lesz használva.

Adatbázis Részletezve

Termékcsoport

Megnevezés Típus Leírás
Category Entitás A termékek csoportosításának entitása, az adatbázis itt tárolja a kategóriák adatait.
id int A termékcsoport ID-je. Ezzel dolgozik a szűrő a termékek csoportosításához. Továbbiak: [SZŰRÉS]
name string A termékcsoport megnevezése. A felhasználó ezt a nevet látja, egy átlagembernek ez az egyértelmű, beazonosítható szöveg.

Termék

Megnevezés Típus Leírás
Product Entitás A termékhez tartozó minden szükséges és listázott adat.
categoryID int A termékcsoport ID-je. Ennek segítségével sorol be egy terméket egy bizonyos termékcsoportba, és így lesz képes az oldal szűrője pontosítani a keresést. Továbbiak: [SZŰRÉS]
name string A termék megnevezése. Egy átlagember által legegyszerűbben olvasható név.
price float A termék ára. Pénznem itt nincsen megadva, azt egységesen a kódban lehet változtatni.
orderAmount int A mennyiség, amelyet a termékből a felhasználó a kosarába kíván tenni.
description string A termék általános leírása. Nem szükséges konkrét adatokat tartalmaznia.
weight string A termék súlya grammokban számolva. A termék saját oldalán található meg.
brand string A termék márkája/készítő cége. A termék saját oldalán található meg.
madeIn string A termék származási országa. A termék saját oldalán található meg.
gender char(1) A termékek fiúra és lányra bontása.
F = Fiú
L = Lány
U = Mindkettő/Unisex
Ez a csoportosítás a szűrést segíti elő. Továbbiak: [SZŰRÉS]
cSize string clothingSize; A termék mérete, hogyha ruházatról van szó; Hogyha játék vagy étel, a termékhez tartozó cSize = 0.
stock int Az érték, ami megadja, hány termék van még a boltban. Hogyha kevesebb elérhető termék van, mint a rendelni kívánt mennyiség, az oldal elutasítja a rendelést.

Rendelés

Megnevezés Típus Leírás
Order Entitás A rendeléshez szükséges adatok entitása. Itt tárolja a rendszer a személyes adatokat és a megrendelni kívánt termékeket.
orderID int A rendelés ID-je. Ezzel az ID-vel lehet beazonosítani és követni a rendelést.
productID int A megrendelt termék ID-je. Ezzel sokkal egyszerűbb követni a megrendeléseket.
uFirstName String A felhasználó keresztneve.
uLastName string A felhasználó vezetékneve.
uEmail string A felhasználó e-mail címe. Erre az email címre kapja a fejleményeket a megrendeléséről.
u1AddressLine string A megrendelő címének első sora.
u2AddressLine string A megrendelő címének második sora.
uPhone string A megrendelő telefonszáma. Ezen a telefonszámon lehet értesíteni a kiszállítás állapotáról.

Felhasználó

Az adatbázisból sok információt megmutat a program, viszont sokmindent - biztonsági és szükségi okokból - nem. Amilyen információkat publikussá tesz a program, azok a felhasználónak szólnak, hogy egyszerűbb legyen neki tájékozódnia és döntést hoznia a vásárlásra nézve.
A következő entitásokból ezek a részek lépnek közvetlen kapcsolatba a felhasználóval:

  1. Category
    1. name
  2. Product
    1. name
    2. price
    3. orderAmount *
    4. description
    5. weight
    6. brand
    7. color
    8. madeIn
    9. cSize
    10. stock
  3. Order
    1. uFirstName *
    2. uLastName *
    3. uEmail *
    4. u1AddressLine *
    5. u2AddressLine *
    6. uPhone *

* :Ezeket az adatokat a felhasználó adja meg, emiatt technikailag közvetlen kapcsolatba lép ezekkel az információkkal.

Szűrés

A szűrés folyamata egy fontos része a weboldalon található termékek közti keresésnek. Ez a funkció segíti elő a gyorsabb vásárlást és a vásárló elégedettségét.
Alapból nincsen bekapcsolva semmilyen szűrő az oldalon, ezeket a felhasználó állíthatja be. Emiatt amikor elkezd böngészni, az összes termék megjelenik egyszerre.
Az oldalon többféle szűrővel képes egyszerre keresni az ember:

  1. Termékcsoportonkénti
  2. Fiú és Lány
  3. Árszerinti
  4. Méret szerinti (ruháknál)
  5. Raktáron lét szerint

Hogyha a szűrőket használva az oldal nem talál megfelelő terméket, erről értesíti a felhasználót.
